# Security Policy

## Supported Versions

This project provides security updates for specific versions. Users are encouraged to upgrade to a supported version to receive security fixes.

| Version | Supported |
| ------- | --------- |
| 5.1.x   | :white_check_mark: Yes |
| 5.0.x   | :x: No   |
| 4.0.x   | :white_check_mark: Yes |
| < 4.0   | :x: No   |

## Reporting a Vulnerability

We take security vulnerabilities seriously. Please follow the steps below to report any issues:

1. **Contact us privately:**  
   Send an email to `hounkpatijosue2020@gmail.com` with a detailed description of the vulnerability, steps to reproduce it, and any relevant files or screenshots.

2. **Acknowledgment:**  
   You will receive a response within 48 hours confirming receipt of your report.

3. **Updates:**  
   We will keep you informed about the progress of the investigation and any patches or mitigations.

4. **Resolution:**  
   Once the vulnerability is confirmed and fixed, you will be credited if you wish, unless you request anonymity.

5. **Responsible Disclosure:**  
   Please do not publicly disclose the vulnerability until a fix has been released.

Thank you for helping us keep this project secure!
